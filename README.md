## 介绍
	这份文档主要用于指导Space相关项目的APIs设计风格，旨在减少共同工作的开发者之间的理解障碍。

	我们强烈推荐开发者们在设计API时遵循这些原则，以防止APIs设计的滥用造成的数量失控和语义混乱。

	这份指导可同时应用于基于HTTP的REST APIs与基于socket的RPC APIs，尤其是gRPC APIs。
